# AI Product Inspector — دليل التشغيل

## الفكرة باختصار
بوت واتساب: التاجر يرسل صورة المنتج + وصف قصير → الذكاء الاصطناعي يفحص
الصورة ويرد بتقرير ملاحظات (تلف، تعارض مع الوصف، جودة الصورة) — **بدون
إصدار حكم نهائي**، فقط ملاحظات تساعد المشتري يقرر.

---

## الخطوات اللي أنت تسويها (15-20 دقيقة إجمالي)

### 1. مفتاح Gemini API (مجاني للبداية)
- روح إلى https://aistudio.google.com/apikey
- سجل دخول بحساب Google → "Create API Key"
- انسخ المفتاح، بتحتاجه بالخطوة 4

### 2. حساب Twilio + WhatsApp Sandbox (مجاني للتجربة)
- سجل في https://www.twilio.com/try-twilio
- من لوحة التحكم روح لـ **Messaging → Try it out → Send a WhatsApp message**
- بيعطونك رقم تجريبي وكود انضمام (مثل "join happy-tiger")
- أرسل الكود من واتسابك الشخصي للرقم عشان تفعّل السانديبوكس

### 3. استضافة الكود (اختر واحد)
أسهل خيار للبداية: **Railway.app**
- سجل بحساب GitHub
- ارفع هذا المجلد على GitHub repo جديد
- في Railway: New Project → Deploy from GitHub → اختر الـ repo
- Railway بيعطيك رابط عام (مثل `https://xxxx.up.railway.app`)

### 4. متغيرات البيئة (Environment Variables)
في إعدادات المشروع على Railway، ضيف:
```
GEMINI_API_KEY=المفتاح_من_الخطوة_1
TWILIO_ACCOUNT_SID=من_لوحة_تويليو
TWILIO_AUTH_TOKEN=من_لوحة_تويليو
```

### 5. ربط الويبهوك
- ارجع للوحة Twilio → WhatsApp Sandbox Settings
- في خانة "When a message comes in" حط:
  `https://xxxx.up.railway.app/webhook/whatsapp`
- احفظ

### 6. اختبار
أرسل صورة أي منتج + وصف قصير على رقم الواتساب التجريبي — لازم يردلك
تقرير خلال ثوانٍ.

## بعد ما يشتغل (الأسبوع الثاني)
- رقم Twilio Sandbox يشتغل بس مع أرقام "انضمت" له — عشان تفتحه لأي حد
  تحتاج ترقية إلى WhatsApp Business API رسمي (يحتاج مراجعة من Meta،
  تاخذ أيام لأسبوع)
- إلى أن توافق ميتا، تقدر تختبر مع أول 10 تجار يدويًا عبر السانديبوكس

## نموذج التسعير المقترح للبداية
- 10 تحليلات مجانية لكل تاجر جديد
- بعدها: اشتراك شهري بسيط بعدد تحليلات محدد
- التتبع (كم تحليل استخدم كل رقم) نضيفه في التحديث القادم بقاعدة بيانات بسيطة
